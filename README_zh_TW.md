# 📝 Meeting Notes Skill

[English](README.md) | **繁體中文**

一個 GitHub Copilot 自訂技能，可將會議逐字稿轉換為專業、結構化的會議紀錄，並自動校正語音辨識錯誤。

## 概述

**Meeting Notes** 技能幫助您將原始會議逐字稿（特別是語音轉文字輸出）轉換為條理分明的會議紀錄。它利用可自訂的詞彙表自動校正術語、人名及縮寫中常見的語音辨識錯誤，然後使用標準化模板進行格式化。

## ✨ 功能特色

- **🔤 自動逐字稿校正** — 使用可編輯的詞彙表修正常見的語音辨識錯誤（例如："欸愛" → "AI"、"開批愛" → "KPI"）
- **👥 與會人員辨識與驗證** — 從逐字稿中提取出席者姓名，並在定稿前向使用者確認
- **📋 標準化模板格式** — 產生一致且專業的會議紀錄，包含結構化的各章節
- **🌐 多語系模板** — 支援英文及繁體中文（zh_TW）會議模板
- **📖 可編輯詞彙表** — 使用者可持續更新術語詞彙表，提升校正準確度

## 🚀 快速開始

### 前置需求

- 支援自訂技能的 [GitHub Copilot](https://github.com/features/copilot)
- VS Code 或相容的編輯器

### 安裝步驟

1. 複製此儲存庫或將技能資料夾複製到您的 Copilot 技能目錄：
   ```bash
   git clone https://github.com/Angus-repo/meeting-notes-skill.git
   ```

2. 自訂 `references/glossary_zh_TW.md` 詞彙表，加入您團隊的：
   - 團隊成員姓名
   - 公司/產品名稱
   - 領域專用術語
   - 常見的語音辨識錯誤

3. 即可搭配 GitHub Copilot 使用！

## 📖 使用方式

### 基本逐字稿處理

向 Copilot 提供原始會議逐字稿，請求產生會議紀錄：

```
請幫我處理這個會議逐字稿：
今天開批愛檢討會議，王曉明報告了欸愛專案的進度...
```

技能將會：

1. **讀取詞彙表** 以了解校正規則
2. **校正語音辨識錯誤** 並呈現校正紀錄
3. **辨識與會人員** 並請您確認
4. **使用模板產生結構化會議紀錄**

### 更新詞彙表

您可以請 Copilot 更新詞彙表以新增術語：

```
請更新詞彙表，新增我們的產品名稱 CloudSync，常被辨識為 Cloud Sink
```

## 📄 模板

| 模板 | 語言 | 檔案 |
|------|------|------|
| English | EN | `assets/meeting-template.md` |
| 繁體中文 | zh_TW | `assets/meeting-template_zh_TW.md` |

兩個模板皆包含以下章節：
- 會議基本資訊（名稱、日期、時間、地點、主持人、記錄人）
- 與會人員名單（出席 / 請假 / 缺席）
- 會議議程
- 討論摘要含決議事項與待辦事項
- 下次會議安排
- 附件與備註

## 🔧 自訂設定

### 詞彙表（`references/glossary_zh_TW.md`）

詞彙表支援以下分類：

| 分類 | 範例 |
|------|------|
| 技術術語 | AI、API、CI/CD、DevOps、Kubernetes |
| 商業術語 | KPI、ROI、SOP、OKR、B2B |
| 部門名稱 | 研發部、業務部、行銷部 |
| 人名 | 中文及英文姓名 |
| 公司/產品名稱 | 自訂項目 |
| 會議相關詞彙 | 議程、決議、共識 |

> **提示：** 當有新團隊成員加入、新專案啟動或發現新的語音辨識錯誤時，請定期更新詞彙表。

## 💡 最佳實踐

1. **保持詞彙表更新** — 更好的詞彙表 = 更高的校正準確度
2. **務必驗證校正結果** — 在定稿前檢閱校正紀錄
3. **確認與會人員名單** — 不要假設自動偵測的名單是完整的
4. **一致使用模板** — 維持所有會議紀錄的專業標準
5. **新增領域專用術語** — 產業術語和內部用語能顯著提升校正效果
